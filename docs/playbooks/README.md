## Playbook の書き方

### ファイル命名規則
```
{番号}-{機能名}-{種類}.md

例:
001-auth-release.md      # 認証機能のリリース
002-db-migration.md      # DBマイグレーション手順
003-rollback.md          # ロールバック手順
```

### 必須セクション

1. **前提条件** - 実行前に必要な準備
2. **手順** - ステップバイステップの実行手順
3. **確認** - 完了後のチェックリスト
4. **ロールバック** - 問題発生時の復旧手順
5. **トラブルシューティング** - よくある問題と解決策

### テンプレート

```markdown
# {タイトル}

## 前提条件
- [ ] 条件1
- [ ] 条件2

## 手順

### 1. ステップ1
\`\`\`bash
コマンド
\`\`\`

### 2. ステップ2
...

## 確認
- [ ] チェック1
- [ ] チェック2

## ロールバック
問題発生時の手順

## トラブルシューティング
### エラー1
原因と解決策
```
